# 库存查询功能说明

## 功能概述
为企业微信移动端开发的库存查询系统，支持通过款号、条码、原厂编号进行商品库存查询。

## 主要特性

### 📱 移动端优化
- 专为手机屏幕设计的响应式界面
- 支持企业微信WebView环境
- 触摸友好的交互设计
- 适配各种移动设备尺寸

### 🔍 多种查询方式
- **款号查询**：支持模糊匹配，快速定位商品
- **条码查询**：精确匹配条码信息
- **原厂编号查询**：支持按厂家编号查询
- **组合查询**：可同时使用多个条件查询

### 📊 智能库存展示
- **库存状态分类**：
  - 🟢 有库存（库存>5）
  - 🟡 低库存（库存≤5且>0）
  - 🔴 缺货（库存=0）

### 🏪 完整商品信息
- 门店名称和地址
- 库存数量
- 商品颜色
- 商品尺寸
- 款号、条码、原厂编号

## API接口设计

### 库存查询接口
```http
GET /api/inventory/query
```
**参数**：
- `styleNumber`: 款号（可选）
- `barcode`: 条码（可选）
- `factoryCode`: 原厂编号（可选）

**响应示例**：
```json
{
  "code": 200,
  "message": "success",
  "data": [
    {
      "id": 1,
      "productName": "男士休闲T恤",
      "styleNumber": "TX001",
      "barcode": "6901234567890",
      "factoryCode": "FC001",
      "color": "白色",
      "size": "L",
      "storeName": "北京朝阳店",
      "storeAddress": "北京市朝阳区三里屯路1号",
      "stockQuantity": 15,
      "minStockThreshold": 5,
      "status": 1
    }
  ]
}
```

## 数据库设计

### 库存表结构
```sql
CREATE TABLE inventory (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    product_name VARCHAR(255) COMMENT '商品名称',
    style_number VARCHAR(100) COMMENT '款号',
    barcode VARCHAR(100) COMMENT '条码',
    factory_code VARCHAR(100) COMMENT '原厂编号',
    color VARCHAR(50) COMMENT '颜色',
    size VARCHAR(50) COMMENT '尺寸',
    store_name VARCHAR(255) COMMENT '门店名称',
    store_address TEXT COMMENT '门店地址',
    stock_quantity INT DEFAULT 0 COMMENT '库存数量',
    min_stock_threshold INT DEFAULT 5 COMMENT '最小库存阈值',
    status TINYINT DEFAULT 1 COMMENT '状态：1-有库存，2-低库存，3-缺货',
    create_time DATETIME DEFAULT CURRENT_TIMESTAMP,
    update_time DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    INDEX idx_style_number (style_number),
    INDEX idx_barcode (barcode),
    INDEX idx_factory_code (factory_code),
    INDEX idx_store_name (store_name)
);
```
